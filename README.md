1. Introducción y descripción
Este proyecto consiste en convertir una Raspberry Pi en un servidor ligero utilizando Docker y Docker Compose como principales herramientas de administración y virtualización. El objetivo principal es crear un sistema económico, eficiente y capaz de ejecutar varios servicios de forma simultánea sin necesidad de utilizar hardware costoso o servidores tradicionales. Gracias al bajo consumo energético y al reducido tamaño de la Raspberry Pi, el sistema puede mantenerse funcionando de forma continua durante las 24 horas del día. Además, Docker permite ejecutar cada servicio dentro de un contenedor independiente, facilitando el mantenimiento, la seguridad y la estabilidad del sistema.

3. Estudio de viabilidad
Antes de comenzar el desarrollo del proyecto se realizó un estudio de viabilidad para analizar si la Raspberry Pi era capaz de soportar correctamente todos los servicios previstos. En este análisis se tuvieron en cuenta factores técnicos, económicos y funcionales. Desde el punto de vista económico, la Raspberry Pi representa una alternativa mucho más barata que un servidor tradicional, tanto por el coste inicial del hardware como por el reducido consumo eléctrico. Técnicamente, Docker permite optimizar los recursos disponibles utilizando contenedores ligeros y aislados, haciendo posible ejecutar múltiples servicios de manera estable y organizada dentro del mismo sistema.

4. Diseño del sistema
El diseño del sistema se basa en una arquitectura modular organizada en distintas capas. La Raspberry Pi actúa como servidor principal, mientras que Raspberry Pi OS Lite proporciona un sistema operativo ligero y optimizado. Sobre este sistema operativo se instala Docker, encargado de ejecutar los servicios dentro de contenedores independientes. Además, Docker Compose permite gestionar toda la configuración del sistema mediante un único archivo, facilitando el despliegue y mantenimiento de los servicios. La estructura modular permite añadir o eliminar aplicaciones fácilmente sin afectar al funcionamiento general del servidor.

5. Implementación
La implementación del proyecto comenzó con la instalación y actualización de Raspberry Pi OS Lite sobre la Raspberry Pi. Posteriormente se instalaron Docker y Docker Compose utilizando los comandos oficiales proporcionados por Docker. Una vez configurado el entorno base, se procedió al despliegue de los distintos servicios mediante archivos docker-compose.yml, configurando puertos, redes y volúmenes persistentes para cada contenedor. Además, antes de realizar la instalación definitiva sobre el hardware real, se realizaron pruebas utilizando QEMU para emular la Raspberry Pi en Windows 10, permitiendo detectar errores y validar configuraciones antes de la implementación final.

6. Configuración de servicios
Durante el desarrollo del proyecto se configuraron distintos servicios utilizando contenedores Docker independientes. Entre ellos destacan WordPress como sistema de gestión de contenidos web, Nextcloud como plataforma de almacenamiento en la nube privada, Plex como servidor multimedia, Duplicati para copias de seguridad automatizadas y Transmission como gestor de descargas torrent. Todos estos servicios fueron desplegados utilizando Docker Compose, facilitando enormemente la administración del sistema y permitiendo mantener una arquitectura organizada, estable y fácilmente ampliable en el futuro.

7. Seguridad
La seguridad fue uno de los aspectos más importantes durante el desarrollo del proyecto. Para proteger el sistema se realizaron actualizaciones periódicas del sistema operativo y se utilizaron imágenes Docker oficiales y verificadas. También se limitaron los puertos expuestos a la red y se mantuvieron los servicios aislados mediante contenedores independientes, reduciendo el riesgo de conflictos o accesos no autorizados. Además, se implementaron copias de seguridad de configuraciones y datos importantes para facilitar la recuperación del sistema en caso de fallos o errores de configuración.

8. Pruebas y resultados
Una vez configurado el servidor, se realizaron distintas pruebas para comprobar el correcto funcionamiento de todos los servicios instalados. Se verificó el inicio automático de los contenedores, el acceso desde dispositivos conectados a la misma red local y la comunicación entre servicios. También se realizaron pruebas de estabilidad, consumo de recursos y persistencia de datos para asegurar que el sistema podía mantenerse funcionando de forma continua sin problemas importantes. Los resultados obtenidos fueron positivos, consiguiendo un servidor estable, funcional y con un consumo energético muy reducido.

9. Conclusiones
El proyecto ha demostrado que una Raspberry Pi puede utilizarse como un servidor ligero, económico y completamente funcional mediante el uso de Docker y Docker Compose. La arquitectura modular basada en contenedores facilita enormemente la administración, mantenimiento y ampliación futura del sistema. Además, el reducido consumo energético y el bajo coste del hardware convierten esta solución en una excelente alternativa para pequeños servidores domésticos, educativos o personales. Desde el punto de vista formativo, el proyecto ha permitido adquirir conocimientos prácticos sobre Linux, Docker, virtualización, redes y administración de sistemas.
